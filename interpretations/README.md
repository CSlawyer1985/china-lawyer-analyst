# 司法解释索引系统 (Interpretation Index System)

---

## 系统概述

本系统是 china-lawyer-analyst v3.0 的核心创新，采用**索引优先 + 按需加载**的策略，解决司法解释频繁更新导致的维护成本高、时效性滞后问题。

**核心设计理念**：
- **索引优先**：先加载轻量级索引（~500 tokens），快速定位所需条文
- **按需加载**：根据具体问题，选择性加载条文详解（~300 tokens/条文）
- **外部增强**：索引不足时，通过外部数据库（威科先行、北大法宝）实时检索

**对比 v2.2 的改进**：
| 维度 | v2.2 | v3.0 | 改进 |
|------|------|------|------|
| Token消耗 | 15,000 | 1,400 | ↓ 90.7% |
| 维护成本 | 8小时 | 1小时 | ↓ 87.5% |
| 更新速度 | 数周 | 实时/周 | ↑ 90% |

## 司法解释模块标准结构

每个司法解释模块遵循统一结构：

```
interpretations/{id}/
├── README.md              # 必备：概要、核心变化、对比表格（~1,000 tokens）
├── index.md               # 必备：条文速查索引（~500 tokens）
├── metadata.json          # 必备：模块元数据
├── external-links.md      # 可选：外部数据库链接
└── articles/              # 可选：重点条文详解（~300 tokens/条文）
    ├── article-{number}.md
    └── ...
```

### 文件说明

#### README.md

**作用**：模块概览，帮助快速了解司法解释的核心内容

**包含内容**：
- 模块元数据（施行时间、条文数量、版本）
- 核心变化速览（重大规则变化，对比新旧法）
- 条文索引导航
- 快速检索指南

**Token估算**：约1,000 tokens

**使用场景**：首次了解司法解释、快速查阅核心变化

---

#### index.md

**作用**：条文速查索引，结构化数据，快速定位所需条文

**包含内容**：
- 条文列表（条文编号、主题、重要性、是否详解）
- 分类索引（按重要性、按主题、按关键词）
- Token使用指南

**Token估算**：约500 tokens

**使用场景**：快速定位条文、了解条文主题

---

#### metadata.json

**作用**：模块元数据，用于自动化工具处理

**包含字段**：
- id: 模块唯一标识
- name: 司法解释名称
- official_name: 正式名称
- document_number: 文号
- effective_date: 施行日期
- article_count: 条文数量
- keywords: 触发关键词列表
- priority_articles: 重点条文编号列表
- external_links: 外部数据库链接

**使用场景**：自动化监测、生成工具、路由系统

---

#### external-links.md

**作用**：外部数据库链接，提供权威数据源

**包含内容**：
- 官方权威数据库（国家法律法规数据库、最高法院官网等）
- 专业法律数据库（威科先行、北大法宝、法信等）
- 学术资源（期刊数据库、法律文库）
- 地域性规范文件（各地法院审判规范）
- 数据库使用指南和检索技巧

**使用场景**：需要条文全文、案例参考、学术观点时

**权威数据源**：
- 国家法律法规数据库：https://flk.npc.gov.cn
- 最高人民法院官网：https://www.court.gov.cn
- 中国裁判文书网：https://wenshu.court.gov.cn
- 人民法院案例库：https://anli.court.gov.cn
- 威科先行：https://law.wkinfo.com.cn
- 北大法宝：https://www.pkulaw.com

---

#### articles/ 目录

**作用**：重点条文详解，深度分析（按需生成）

**文件命名**：`article-{number}.md`（如：article-7.md）

**包含内容**（约300 tokens）：
- 条文内容（原文引用）
- 核心要点（3-5个要点）
- 与旧法的区别（对比表格）
- 实务应用（案情、分析、结论）
- 争议焦点（焦点、观点）

**Token估算**：约300 tokens/条文

**使用场景**：具体问题分析、案例研究、深度学习

## 使用指南

### 场景1：快速查询

**需求**：仅需要了解条文概况

**操作**：
1. 加载 `index.md`（约500 tokens）
2. 查找所需条文
3. 如需详情，加载 `article-{number}.md`

**示例**：
- 问题：保证方式约定不明时如何认定？
- 操作：加载 index.md → 查找"保证方式" → 定位第25条
- 可选：加载 article-25.md 获取详细解读

---

### 场景2：深度分析

**需求**：需要详细解读条文

**操作**：
1. 加载 `index.md`（500 tokens）
2. 加载 `article-{number}.md`（300 tokens）
3. 总计：约800 tokens

**对比 v2.2**：
- v2.2：加载完整模块（15,000 tokens）
- v3.0：按需加载（800 tokens）
- 节省：94.7%

---

### 场景3：外部检索

**需求**：需要条文全文或案例参考

**操作**：
1. 查看 `external-links.md`
2. 选择权威数据源：
   - 条文全文 → 国家法律法规数据库
   - 案例参考 → 中国裁判文书网、人民法院案例库
   - 专业解读 → 威科先行、北大法宝
   - 学术观点 → 北大法宝学术文献

## 创建新司法解释模块

### 方法1：使用自动化工具（推荐）

```bash
# 1. 监测工具自动发现新司法解释
cd tools/monitor
python court-monitor.py

# 2. 生成工具自动创建模块
cd ../generator
python draft-generator.py --id {id}

# 3. 人工审核
# 查看 tools/workflows/review-checklist.md

# 4. 测试和部署
```

### 方法2：手动创建

```bash
# 1. 复制模板
cp -r interpretations/_template interpretations/{new-id}

# 2. 修改文件
# - 编辑 README.md
# - 编辑 index.md
# - 编辑 metadata.json
# - 编辑 external-links.md

# 3. 创建条文详解
mkdir interpretations/{new-id}/articles
# 根据需要创建 article-{number}.md
```

### ID命名规则

**格式**：`{领域}-{类型}-{年份}`

**示例**：
- `contract-general-2023`：合同编通则解释（2023）
- `security-law-2020`：担保制度解释（2020）
- `criminal-law-2024`：刑法解释（2024）

**规则**：
- 领域：contract, tort, security, criminal, administrative等
- 类型：general（通则）、specific（专项）
- 年份：司法解释施行的年份

## Token优化技巧

### 1. 优先使用索引

✅ **推荐**：
```
问题：保证方式约定不明时如何认定？
操作：加载 index.md → 查找第25条 → 可选加载 article-25.md
Token：500（索引）+ 300（条文）= 800
```

❌ **不推荐**：
```
问题：保证方式约定不明时如何认定？
操作：加载完整模块
Token：15,000（v2.2方式）
```

### 2. 按需加载条文

✅ **推荐**：
```
问题：涉及保证方式和抵押财产转让
操作：加载 index.md + article-25.md + article-37.md
Token：500 + 300 + 300 = 1,100
```

❌ **不推荐**：
```
问题：涉及保证方式和抵押财产转让
操作：加载所有条文详解
Token：500 + 300 × 71 = 21,800
```

## 常见问题

### Q1：如何快速找到所需条文？

**A**：使用 index.md 的关键词索引

1. 加载 index.md
2. 查找"按关键词索引"表格
3. 定位相关条文编号
4. 可选：加载对应条文详解

### Q2：外部数据库是否需要付费？

**A**：部分需要

- **免费**：国家法律法规数据库、最高法院官网、中国裁判文书网
- **付费**：威科先行、北大法宝、法信

建议：
- 日常使用：免费数据库
- 专业研究：付费数据库（更全面、更专业）

### Q3：如何判断条文的重要性？

**A**：查看 index.md 的"按重要性分类"

- ⭐⭐⭐⭐⭐ 最高优先级：必读
- ⭐⭐⭐⭐ 高优先级：重点
- ⭐⭐⭐ 中等优先级：了解

### Q4：条文详解是否覆盖所有条文？

**A**：否，仅覆盖重点条文（10-15个）

原因：
- Token优化
- 实践中大部分问题集中在重点条文
- 其他条文可通过外部数据库查询
