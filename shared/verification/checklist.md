# 验证清单 (Verification Checklist)

---

## 模块说明

本模块是 china-lawyer-analyst v2.0 的共享验证工具，提供法律分析的10项验证标准，用于验证分析的准确性和完整性。

**适用领域**：所有领域

**依赖模块**：shared/verification/rubric.md

**触发场景**：完成分析后验证准确性时

---

## 内容

### 一、10项验证标准

#### 1. 法律条文验证

**验证项**：
- [ ] 所有法律条文引用准确（条文号、内容）
- [ ] 司法解释引用现行有效
- [ ] 指导案例编号正确
- [ ] 法律条文未失效或被废止
- [ ] 引用了最新修正版法律

**验证方法**：
- 威科先行、北大法宝查询条文是否现行有效
- 查看法律的施行时间、修正时间
- 查看司法解释的发布时间、是否失效

#### 2. 案例事实验证

**验证项**：
- [ ] 事实描述完整、准确
- [ ] 事实描述有证据支持
- [ ] 争议焦点识别准确
- [ ] 关键事实未遗漏
- [ ] 事实描述客观中立

**验证方法**：
- 回顾案件材料（合同、证据、当事人陈述）
- 核对证据清单
- 确认事实描述与证据一致

#### 3. 法律适用验证

**验证项**：
- [ ] 法律适用准确（成文法、司法解释）
- [ ] 法条与案件事实匹配
- [ ] 法律适用逻辑严密
- [ ] 考虑了类案裁判规则
- [ ] 区分了强制性规定与任意性规定

**验证方法**：
- 使用 IRAC 方法重新分析
- 检查大前提（法条）、小前提（事实）、结论是否一致
- 参考类案检索结果

#### 4. 程序法验证

**验证项**：
- [ ] 管辖法院正确（级别管辖、地域管辖）
- [ ] 诉讼时效未届满
- [ ] 除斥期间未届满
- [ ] 程序阶段识别准确
- [ ] 程序选项考虑全面

**验证方法**：
- 民事诉讼法第17-35条（管辖）
- 民法典第188条（诉讼时效）
- 相关司法解释（诉讼时效、除斥期间）

#### 5. 举证责任验证

**验证项**：
- [ ] 举证责任分配正确
- [ ] 考虑了举证责任倒置
- [ ] 证据清单完整
- [ ] 证明标准明确（高度可能性、排除合理怀疑）
- [ ] 举证期限考虑

**验证方法**：
- 民诉法解释第90-108条（举证责任）
- 民诉法解释第108条（证明标准）
- 民诉法解释第99条（举证期限）

#### 6. 救济措施验证

**验证项**：
- [ ] 救济措施合法（继续履行、赔偿损失、违约金、定金）
- [ ] 损害赔偿计算准确
- [ ] 违约金是否过高或过低
- [ ] 考虑了减损义务
- [ ] 考虑了过失相抵

**验证方法**：
- 民法典第577条（违约责任）
- 民法典第584条（损害赔偿）
- 民法典第585条（违约金调整）
- 民法典第591条（减损义务）
- 民法典第1173条（过失相抵）

#### 7. 风险评估验证

**验证项**：
- [ ] 胜诉概率评估合理
- [ ] 可能结果分析全面（最佳、最差、最可能）
- [ ] 诉讼成本估算准确
- [ ] 时间成本考虑
- [ ] 声誉风险、关系风险考虑

**验证方法**：
- 参考类案检索结果
- 考虑当事人证据强弱
- 考虑法律适用的不确定性
- 考虑法官自由裁量权

#### 8. 伦理和合规验证

**验证项**：
- [ ] 法律建议符合律师职业伦理
- [ ] 未提供虚假法律意见
- [ ] 未鼓励违法行为
- [ ] 提示了诉讼风险
- [ ] 建议咨询专业律师

**验证方法**：
- 律师执业行为规范
- 律师职业伦理
- 法律法规强制性规定

#### 9. 引用格式验证

**验证项**：
- [ ] 法律条文引用格式正确
- [ ] 案例引用格式正确
- [ ] 司法解释引用格式正确
- [ ] 参考资料标注完整
- [ ] 引用来源权威可靠

**验证方法**：
- 参考本文档"引用来源标准化方案"
- 确保所有引用都有明确来源

#### 10. 语言表达验证

**验证项**：
- [ ] 法律术语使用准确
- [ ] 语言表达清晰、简洁
- [ ] 逻辑严密、条理清晰
- [ ] 无错别字、语法错误
- [ ] 符合法律文书规范

**验证方法**：
- 逐句阅读检查
- 使用专业术语词典
- 参考法律文书范本

---

### 二、验证流程

#### 2.1 自我验证流程

**步骤1：完成分析**
- 完成法律分析初稿

**步骤2：逐项验证**
- 按照验证清单逐项验证
- 标注验证结果（通过/不通过）

**步骤3：修正问题**
- 对不通过的项目进行修正
- 重新验证

**步骤4：最终确认**
- 所有项目验证通过
- 完成最终版本

#### 2.2 同行评审流程

**步骤1：提交分析**
- 提交法律分析初稿

**步骤2：同行评审**
- 同事按照验证清单逐项验证
- 提供评审意见

**步骤3：修正问题**
- 根据评审意见修正问题

**步骤4：最终确认**
- 评审通过
- 完成最终版本

### 四、常见验证问题

#### 4.1 法律条文问题
- 条文号错误
- 条文内容引用不准确
- 条文已失效或被废止
- 未引用最新修正版

#### 4.2 案例事实问题
- 事实描述不完整
- 事实描述与证据不一致
- 争议焦点识别不准确
- 关键事实遗漏

#### 4.3 法律适用问题
- 法律适用错误
- 法条与案件事实不匹配
- 法律适用逻辑不严密
- 未考虑类案裁判规则

#### 4.4 程序法问题
- 管辖法院错误
- 诉讼时效已届满
- 除斥期间已届满
- 程序阶段识别错误

#### 4.5 举证责任问题
- 举证责任分配错误
- 未考虑举证责任倒置
- 证据清单不完整
- 证明标准不明确

### 五、快速参考

| 验证类别 | 核心验证项 | 常见问题 |
|---------|-----------|---------|
| 法律条文 | 条文号、内容、效力 | 条文错误、已失效 |
| 案例事实 | 完整性、准确性、客观性 | 事实遗漏、与证据不符 |
| 法律适用 | 准确性、匹配性、逻辑性 | 适用错误、逻辑漏洞 |
| 程序法 | 管辖、时效、期间 | 管辖错误、时效届满 |
| 举证责任 | 分配、倒置、标准 | 分配错误、标准不明 |
