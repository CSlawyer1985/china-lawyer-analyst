# 共享模块说明文档 (Shared Modules Documentation)

---

## 模块概述

本目录（shared/）包含 china-lawyer-analyst v2.0 的共享模块，这些模块按需加载，可跨领域复用。

**模块特点**：
- **模块化设计**：每个文件独立、完整
- **跨领域复用**：可被多个领域模块引用
- **格式统一**：使用相同的文件头部模板
- **内容完整**：包含相关的方法、资源、工具

---

## 使用说明

### 按需加载

共享模块采用按需加载机制，只在需要时加载相关模块：

**示例1：需要检索法律条文**
```
加载：shared/methods/legal-research.md
加载：shared/resources/databases.md
```

**示例2：需要起草法律文书**
```
加载：shared/methods/legal-writing.md
加载：shared/resources/templates.md
```

**示例3：需要验证分析质量**
```
加载：shared/verification/rubric.md
加载：shared/verification/checklist.md
加载：shared/verification/pitfalls.md
```

### 跨领域复用

共享模块可被多个领域模块引用：

**示例：公司法领域**
```
领域模块：domains/companies/
依赖共享模块：
  - shared/methods/legal-research.md
  - shared/methods/legal-writing.md
  - shared/methods/due-diligence.md
  - shared/resources/databases.md
  - shared/verification/rubric.md
  - shared/verification/checklist.md
  - shared/verification/pitfalls.md
```

**示例：建筑工程领域**
```

依赖共享模块：

  - shared/methods/negotiation.md
```

## 常见问题

### Q1：为什么需要共享模块？

**A**：共享模块实现了代码复用，避免重复内容，提高维护效率。

### Q2：如何添加新的共享模块？

**A**：按照以下步骤添加：
1. 确定模块类型（方法/资源/验证工具）
2. 在对应目录创建新文件
3. 使用统一的文件头部模板
4. 填写模块内容
5. 更新本 README.md 文件

### Q3：如何修改现有模块？

**A**：按照以下步骤修改：
1. 识别需要修改的模块
2. 进行修改（保持格式统一）
3. 更新版本信息
4. 通知相关领域模块维护者

### Q4：共享模块与领域模块有什么区别？

**A**：
- **共享模块**：通用性强，可跨领域复用
- **领域模块**：针对特定领域，包含领域特定的法律、法规、案例、实务

### Q5：如何确定一个内容应该放在共享模块还是领域模块？

**A**：
- **共享模块**：如果内容适用于2个及以上领域
- **领域模块**：如果内容仅适用于1个领域
