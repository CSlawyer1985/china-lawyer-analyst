## 项目简介

`china-lawyer-analyst` 是一个适用于中国大陆法律体系的专业法律分析 Claude Skill。该 skill 将美国普通法系的 `lawyer-analyst` 本土化改造，基于大陆法系（成文法）体系，运用中国法律方法论进行法律分析。

### v3.1.0 核心升级

**混合架构 v3.1**：
- ✅ **司法解释索引系统**：支持按需加载条文详解，维护成本降低 87.5%
- ✅ **自动化工具集**：版本检查、增强检索、法院监测、定时任务配置
- ✅ **智能三级路由**：核心规则 → 司法解释检索 → 动态 MCP 工具调用
- ✅ **Token 优化**：平均节省 56% tokens（v2.0 基础上进一步优化）
- ✅ **实时数据源**：集成 Gety MCP + Unifuncs + 12个官方数据库

**司法解释模块**：
- ✅ 合同编通则解释（2023）
- ✅ 安全法解释（2020）
- ✅ 可扩展模板系统（轻松添加新司法解释）

## 快速上手

### 1. Skill 加载

将 `china-lawyer-analyst` 放入 Claude Code 的 skills 目录：

```bash
cp -r china-lawyer-analyst ~/.claude/skills/
```

### 2. 基础使用

在 Claude Code 中调用 skill：

```
请使用 china-lawyer-analyst skill 分析以下法律问题：
[粘贴案件事实或法律问题]
```

### 3. 工作原理

**混合架构 v3.1 - 三级智能路由**：

**Level 1: 核心规则路由**
- 自动识别法律领域（合同法、侵权法、建设工程等）
- 加载对应的核心模块 + 领域模块

**Level 2: 司法解释检索**
- 识别问题涉及的司法解释需求
- 按需加载具体司法解释条文详解
- 动态索引，支持精准匹配

**Level 3: 动态 MCP 工具调用**
- 集成 Gety MCP（本地文档检索）
- 集成 Unifuncs（网络搜索）
- 集成 11个官方权威数据库（国家法律法规数据库、最高法院官网、裁判文书网、人民法院案例库等）
- 实时检索最新法规、案例、实务文章
- **优先使用官方免费数据库，不依赖付费专业数据库**

**Token 优化效果**（v3.1 vs v1.0）：
- **单领域问题**：节省 **56%** tokens（从 37,958 降至 ~16,700）
- **多领域问题**：节省 **48%** tokens（从 37,958 降至 ~19,800）
- **边界问题**：节省 **45%** tokens（从 37,958 降至 ~20,900）

**维护成本优化**：
- 司法解释索引系统：维护成本降低 **87.5%**
- 按需加载，无需重复更新核心文件
- 模板化扩展，轻松添加新司法解释

## 典型使用场景

### 场景 A：合同纠纷（涉及司法解释）

```
请使用 china-lawyer-analyst skill 分析：

甲乙公司签订软件开发合同，约定2023年6月30日前交付。
乙方延迟交付1.5周，且软件存在缺陷。甲方拒绝付款。

涉及问题：预约合同、违约责任、合同解除
```

**系统行为**：
- 识别领域：`contract-law` + `contract-interpretation-2023`
- 加载模块：核心模块 + 合同法领域 + 合同编通则解释（2023）
- 路由策略：
  - Level 1: 加载合同法领域模块
  - Level 2: 检索《最高人民法院关于适用〈中华人民共和国民法典〉合同编通则若干问题的解释》相关条文
  - Level 3: 如需，调用威科先行检索相关案例
- Token 消耗：~18,500 tokens（节省 51%）

```
医疗费5万元、误工费3万元。监控显示地面有水渍、无警示标识。

请分析：商场是否违反安全保障义务？是否应当承担侵权责任？
```

**系统行为**：
- 识别领域：`tort-law`
- 加载模块：核心模块 + 侵权法领域
- Token 消耗：~16,700 tokens（节省 56%）

### 场景 C：担保纠纷（涉及司法解释）

```
请使用 china-lawyer-analyst skill 分析：

甲公司向乙银行借款1000万元，丙公司提供保证担保。
保证合同未约定保证方式。借款到期后，甲公司未还款。

乙银行直接起诉丙公司要求承担保证责任。

问题：丙公司是一般保证还是连带责任保证？是否享有先诉抗辩权？
```

## 10步法中国法律分析流程

当使用 `china-lawyer-analyst` skill 时，系统将遵循以下10步流程：

### Step 1: 收集事实
- 识别关键事实
- 整理时间线
- 识别当事人

### Step 2: 识别法律问题
- 将事实转化为法律问题
- 识别争议焦点

### Step 3: 研究适用法律
- 查找成文法（法律、行政法规）
- **[v3.1新增]** 查找司法解释（按需加载索引）
- 参照指导案例

### Step 4: IRAC分析
- **Issue**：问题识别
- **Rule**：规则识别（含司法解释）
- **Application**：适用分析
- **Conclusion**：结论

### Step 5: 识别诉讼请求和抗辩
- 确定我方诉讼请求（原告）或抗辩理由（被告）
- 评估证据支持

### Step 6: 评估程序阶段和选项
- 确定当前程序阶段（起诉前、一审、二审、再审、执行）
- 评估程序选项（财产保全、证据保全）

### Step 7: 评估风险和可能结果
- 胜诉概率评估
- 可能结果分析（最佳、最差、最可能）
- 成本收益分析

### Step 8: 识别和评估救济措施
- 继续履行、赔偿损失、违约金、定金
- 量化损害赔偿

### Step 9: 考虑和解和替代争议解决
- 评估 BATNA（最佳替代方案）
- 考虑人民调解、司法调解、商事仲裁

### Step 10: 提供法律建议和行动计划
- 综合分析，提供明确法律意见
- 提出策略建议
- 提出行动计划
